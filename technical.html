Background Information

A Perfect Clear (PC) is where the Tetris board becomes empty again, usually after placing 10 pieces.

There is 7 tetrominos/pieces, which are given to players in groups called bags, meaning that you're guranteed each piece within each bag.

Since a PC is usually 10 pieces, but there are 7 pieces per bag, this leads to the subsequent PC starts off with part of a bag.

Example: On 2nd PC, after placing 7 pieces then 3 pieces for 1st PC, you're left with 4 pieces from that 2nd bag at start of the 2nd PC.

These pieces leftover are called saves from the previous PC.

In modern tetris, a hold allows the player to hold a piece for later use. This hold allows for some control on the save to the next PC.

Usually when doing PCs, one would make a setup with 3 or 4 pieces then solve with the remaining 7 pieces that can be seen.

hold-generalizer

A professional software engineering focused project with the intent of simulating if a user was able to hold some n number of pieces rather than the normal one piece.

This program is able to generate the necessary data to extract chance to build/solve a setup and chance for various saves.

Underlying logic description

A term I use is a n-hold permutation after looking into how multiple holds would work.

By swapping pieces n distance away from the original location to the right, this produces queues that are equivalent to having the ability hold n pieces.

Using these queues without holding to try to solve or build setups will produce valid results, which is how this program is mostly based on.
